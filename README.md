# PrismBench

PrismBench یک پنل محصولی PHP برای تست و فروش پیام‌های چندمدلی است.

🌐 **نسخهٔ آنلاین:** https://onwebs.ir/hesabi/

> ⚠️ **استفاده بدون ذکر منبع مجاز نیست.** کلیه حقوق متعلق به **سپهر فتحی** است. هرگونه استفاده، کپی یا راه‌اندازی این پروژه نیازمند اجازهٔ کتبی نویسنده و ذکر نام نویسندهٔ اصلی است. جزئیات در [LICENSE](LICENSE).

## قابلیت‌ها

- installer شبیه `hosho` در `/install/` برای اتصال MySQL و ساخت ادمین
- ذخیره دیتابیسی settings، admins، users، wallet، conversations، logs و payments
- آرنا side-by-side برای اجرای ۱ تا ۴ مدل همزمان
- حالت‌های عمومی، کدنویسی، بنچمارک و RAG
- کیف‌پول، اعتبار اولیه، OTP تستی، تاریخچه گفتگو و شارژ واقعی با Zibal/ZarinPal
- قیمت‌گذاری با نرخ provider، کارمزد درصدی، هزینه ثابت هر درخواست، هزینه ثابت هر مدل موفق و حداقل شارژ هر مدل
- غیرفعال‌سازی provider از UI ادمین و غیرفعال/مخفی کردن مدل‌ها از JSON کاتالوگ با `enabled` و `access`
- بکاپ/بازیابی دیتابیس از پنل ادمین
- صفحه عمومی benchmark/cost بدون لینک ادمین برای کاربر

## اجرا در توسعه

```bash
/Applications/MAMP/bin/php/php8.3.14/bin/php -S 127.0.0.1:8000
```

مسیرها:

- نصب: `http://127.0.0.1:8000/install/`
- محصول: `http://127.0.0.1:8000/`
- بنچمارک: `http://127.0.0.1:8000/benchmarks.php`
- ادمین: `http://127.0.0.1:8000/adminpanel/`

## نصب

در `/install/` اطلاعات MySQL، ادمین اولیه، کلیدهای Yarabot/GapGPT/Arvan و تنظیمات قیمت‌گذاری را وارد کنید. installer فایل `config.php` را می‌سازد و جدول‌های `pa_*` را ایجاد می‌کند.

اگر خواستید دستی نصب کنید، از `config.sample.php` کپی بگیرید و بعد `pl_db_init()` را از installer اجرا کنید.

## کنترل دسترسی مدل‌ها

در پنل ادمین، تب `Provider و مدل`:

- provider با checkbox فعال/غیرفعال می‌شود.
- داخل `providers_json` برای هر مدل:
  - `enabled: "0"` مدل را کاملاً غیرفعال می‌کند.
  - `access: "public"` مدل را در UI کاربر نشان می‌دهد.
  - `access: "private"` مدل را از لیست عمومی خارج می‌کند ولی برای فراخوانی مستقیم نگه می‌دارد.
  - `access: "admin"` یا `disabled` مدل را از دسترس کاربر خارج می‌کند.

## بکاپ

از تب `بکاپ` در ادمین می‌توانید بکاپ دیتابیس بگیرید یا بکاپ قبلی را بازیابی کنید. اگر PHP `ZipArchive` فعال باشد خروجی ZIP است؛ اگر فعال نباشد خروجی SQL مستقیم دانلود می‌شود. restore هر دو حالت `.zip` و `.sql` را می‌پذیرد، اما فقط dump جدول‌های داخلی `pa_*` را اجرا می‌کند و بعد از بازیابی migrationهای لازم را دوباره اعمال می‌کند.

## Production

در production:

- `otp_debug_enabled` را خاموش کنید.
- ارسال SMS را در پنل ادمین فعال و یکی از حالت‌های `melipayamak` یا `webhook` را کامل تنظیم کنید؛ در غیر این صورت با خاموش بودن OTP تستی، ورود کاربران متوقف می‌شود.
- `topup_sandbox_enabled` را خاموش نگه دارید، `payment_enabled` را روشن کنید و Merchant درگاه Zibal یا ZarinPal را در پنل ادمین وارد کنید.
- callbackهای واقعی آماده‌اند: `/payment/zibal_callback.php` و `/payment/zarinpal_callback.php`.
- دسترسی مستقیم به `data/` را در وب‌سرور deny کنید.
